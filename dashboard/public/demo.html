 Act 2 of 5

402 Without Payment

Hit the gateway without a payment header. You get a 402 Payment Required — like HTTP 401 but for money. The gateway enforces payment before proxying.

🎯 The point: Without payment, agents are blocked. The response tells the agent exactly how much to pay, on which network, and to which address — all machine-readable.

Run this in your terminal

$ curl -s http://localhost:4402/api/v1/posts | python3 -m json.tool

Expected Response — HTTP 402

Gateway Response
HTTP/1.1 402 Payment Required Content-Type: application/json
{   "x402Version": 1,   "accepts": [     {       "scheme": "exact",       "network": "eip155:84532",       "maxAmountRequired": "1000",       "payTo": "0x3E05...C1a8",       "asset": "0x036Cbd...USDC"     }   ],   "error": "Payment required" }
💡 Key insight: The 402 response is fully machine-readable. An AI agent reads maxAmountRequired, payTo, and network — then signs and sends the payment automatically. No human needed.
 Act 3 of 5

Fund the Agent Wallet

The SDK creates a fresh CDP wallet on init(). Fund it with test USDC on Base Sepolia so the agent can pay for requests autonomously.

1
Run the agent to get its wallet address
The agent creates a new CDP wallet each time. Run it once — it will print the wallet address before failing (no funds yet).
2
Copy the agent wallet address from the output
Look for a line like: Agent wallet: 0xAbCd...1234
3
Get test USDC from Circle Faucet
Go to faucet.circle.com → select Base Sepolia → paste the agent wallet address → request USDC.
4
Wait ~30 seconds for funds to arrive
Base Sepolia is fast. You can check the balance on sepolia.basescan.org by searching the agent address.

Agent init — what the SDK does

examples/agent-demo/run.ts
// SDK creates a CDP wallet and wraps fetch with x402 auto-payment const client = new RoutefiClient({ gatewayBaseUrl: "http://localhost:4402" });
await client.init(); // ↑ Creates CDP account, registers EVM payment scheme, wraps fetch
✓ Agent wallet: 0xAbCd...1234 ← fund this address
⚠️ Note: CDP creates a new wallet address each run. Fund the address from the current run's output. For a persistent wallet, you'd serialize and reuse the account — not needed for this demo.
